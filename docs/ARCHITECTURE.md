## 模块详解

### 1. 身份认证模块 (Auth Module)

**流程图:**
```
┌─────────────┐
│  用户输入   │
│ (用户名/密码)│
└──────┬──────┘
       │
┌──────▼───────────────┐
│ 密码加密 (bcrypt)    │
└──────┬───────────────┘
       │
┌──────▼───────────────┐
│ 数据库验证           │
└──────┬───────────────┘
       │
    ┌──┴──┐
    │     │
    YES   NO
    │     │
┌───▼──┐ ┌─▼──────────┐
│生成  │ │返回错误    │
│JWT   │ │(401)       │
└───┬──┘ └────────────┘
    │
┌───▼──────────────┐
│返回 Token        │
│+ 用户信息        │
└──────────────────┘
```

**Token 结构:**
```json
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "userId": "user_id",
    "username": "username",
    "email": "email@example.com",
    "iat": 1234567890,
    "exp": 1234654290
  },
  "signature": "HMACSHA256(encoded_header + encoded_payload + secret)"
}
```

### 2. 实时通讯模块 (Socket.io)

**连接流程:**
```
┌─────────────┐
│ 客户端连接  │
└──────┬──────┘
       │
┌──────▼───────────────┐
│ 服务器接收连接       │
└──────┬───────────────┘
       │
┌──────▼───────────────┐
│ 验证 JWT Token       │
└──────┬───────────────┘
       │
    ┌──┴──┐
    │     │
   有效   无效
    │     │
┌───▼──┐ ┌─▼─────────┐
│加入  │ │断开连接   │
│房间  │ │(401)      │
└───┬──┘ └───────────┘
    │
┌───▼─────────────────┐
│监听客户端事件       │
│(发送消息、输入等)   │
└─────────────────────┘
```

**事件类型:**
- **Chat Events**: 
  - `send-message` - 发送消息
  - `receive-message` - 接收消息
  - `typing` - 正在输入
  
- **Friend Events**:
  - `friend-request` - 好友请求
  - `friend-accepted` - 请求被接受
  - `friend-online/offline` - 上下线

- **Notification Events**:
  - `notification` - 接收通知
  - `notification-read` - 通知已读

### 3. 数据持久化 (MongoDB)

**集合结构:**
```
dongming-community
├── users (用户集合)
│   └── indexes: username, email
├── friendRequests (好友请求)
│   └── indexes: toUserId, status
├── chats (聊天)
│   └── indexes: user1Id, user2Id
├── messages (消息)
│   └── indexes: chatId, createdAt
├── posts (动态)
│   └── indexes: userId, createdAt
├── comments (评论)
│   └── indexes: postId, userId
├── advertisements (广告)
│   └── indexes: category, status, price
└── notifications (通知)
    └── indexes: userId, isRead
```

### 4. 缓存层 (Redis)

**缓存策略:**
```
┌──────────────┐
│   请求数据   │
└──────┬───────┘
       │
┌──────▼──────────┐
│检查 Redis 缓存  │
└──────┬──────────┘
       │
    ┌──┴──┐
    │     │
   HIT   MISS
    │     │
┌───▼──┐ ┌─▼─────────────────┐
│返回  │ │查询数据库          │
│缓存  │ └─────┬──────────────┘
│数据  │       │
└────┘  ┌──────▼──────────────┐
        │数据写入 Redis       │
        │(设置过期时间)       │
        └─────┬──────────────┘
              │
        ┌─────▼──────────────┐
        │返回数据给客户端    │
        └────────────────────┘
```

**缓存 Key 设计:**
```
user:{userId}                    - 用户信息
chat:{chatId}:messages           - 聊天消息
post:{postId}                    - 动态信息
ad:{adId}                        - 广告信息
online_users                     - 在线用户列表
friend_requests:{userId}         - 好友请求列表
```

## 通信协议

### REST API 调用流程

```
客户端                           服务器
  │                                │
  ├──── HTTP Request ────────────► │
  │     (GET/POST/PUT/DELETE)     │
  │                                │
  │                              ┌─▼─────────┐
  │                              │ 中间件验证 │
  │                              │ (JWT等)   │
  │                              └─┬─────────┘
  │                                │
  │                              ┌─▼──────────┐
  │                              │控制器处理  │
  │                              │(业务逻辑)  │
  │                              └─┬──────────┘
  │                                │
  │                              ┌─▼──────────┐
  │                              │数据库操作  │
  │                              └─┬──────────┘
  │                                │
  │  ◄────── HTTP Response ────── │
  │   (JSON + Status Code)        │
  │                                │
```

### WebSocket 双向通信

```
客户端                           服务器
  │                                │
  ├───── WebSocket Connect ────────►│
  │                                │
  │◄─── Connection Established ────┤
  │                                │
  ├───── Send Message ────────────►│
  │  (event: 'send-message')      │
  │                                │
  │                              ┌─▼─────────┐
  │                              │处理消息   │
  │                              │保存DB     │
  │                              └─┬─────────┘
  │                                │
  │◄──── Receive Message ────────┤ │
  │  (event: 'receive-message')    │
  │                                │
  ├─────── Emit Event ────────────►│
  │  (typing, read, etc.)         │
  │                                │
  │◄────── Broadcast Event ──────┤ │
  │  (to other clients)            │
```

## 安全设计

### 1. 身份验证与授权
```
请求 → JWT 验证 → 权限检查 → 业务逻辑 → 响应
       (authenticate)  (authorize)
```

### 2. 数据加密
- 密码: bcrypt (10-12 rounds)
- 敏感数据: AES-256
- HTTPS: SSL/TLS 加密

### 3. 防护措施
- CORS 跨域控制
- Rate Limiting 限流
- SQL Injection 防护 (Mongoose)
- XSS 防护 (Content Security Policy)
- CSRF 令牌验证
